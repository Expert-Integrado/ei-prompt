## Phase Goal

**As a** developer creating a client project via `/ei-cria-cliente`, **I want to** go through three auditable, gated steps (scaffold folders → collect all required fields incl. media → fill templates) with a hard confirmation gate before filling, **so that** no client is ever left with silently incomplete required fields.

<objective>
Prove, via a regression test and two live sessions, that the full three-step gated flow built in Plans 02-01 through 02-04 actually behaves as specified end-to-end — in both `single-agent` and `multi-agente-especialidade-unica` modes, including a deliberate attempt to bypass the hard gate. Per `02-VALIDATION.md`, this is a manual-only verification surface (no test framework exists for prompt/subagent conversational behavior in this repo); this plan is the phase's designated proof point, mirroring Phase 1's own precedent for orchestration/hook behavior (`01-03-PLAN.md`'s live human-verify task).

Purpose: every other plan in this phase produced static file content; nothing has yet exercised the actual dispatch chain, the actual confirmation gate, or the actual hook trigger in a live Claude Code session.
Output: a green regression run of Phase 1's test suite, plus two recorded human-verify outcomes covering both modes and the Cancel path.
</objective>
